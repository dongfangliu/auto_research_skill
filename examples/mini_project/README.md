# Mini Project Example

This example demonstrates the MVP workflow with a small fictional research trail.

The topic is deliberately lightweight: whether an agent-assisted research notebook improves recovery after task interruption. The goal is not to make a real scientific claim, but to show how ideas, literature questions, hypotheses, experiments, evidence, claims, and manuscript outlines connect.

For a daily continue, read only:

1. `PROJECT_PROFILE.md`
2. `STATE.md`
3. the primary active path listed in state, currently `experiments/EXP-0002_brief.md`

Do not read `TEAM_PROFILE.md` for ordinary daily continue. Read it only for team/advisor requests, experiment readiness review, claim audit, or manuscript outline review.

Read `syntheses/SYN-0001.md` only when the task asks for experiment branch synthesis, evidence candidates, claim warnings, or when `STATE.md` says the latest synthesis is needed to recover recent experiment-thread understanding.

For full audit, read in this order:

1. `PROJECT_PROFILE.md`
2. `TEAM_PROFILE.md`
3. `STATE.md`
4. `ideas/IDEA-0001.md`
5. `literature/LQ-0001.md`
6. `literature/LIT-0001.md`
7. `hypotheses/HYP-0001.md`
8. `experiments/EXP-0001_brief.md`
9. `experiments/EXP-0001_decision.md`
10. `ideas/IDEA-0002.md`
11. `hypotheses/HYP-0002.md`
12. `experiments/EXP-0002_brief.md`
13. `handoffs/HANDOFF-0001.md`
14. `evidence/EVD-0001.md`
15. `syntheses/SYN-0001.md`
16. `claims/CLM-0001.md`
17. `manuscripts/OUTLINE-0001.md`
